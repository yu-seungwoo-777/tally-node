## 기술 접근 방식

### 문서 형식

**Markdown 표 사용**:
```markdown
### 하드웨어 사양

| 항목 | 사양 | 설명 |
|------|------|------|
| MCU | ESP32-S3 | Xtensa LX7 @ 240MHz |
| Flash | 4MB | 펌웨어 저장 |
```

**기능 상태 표시**:
- ✅ 지원 (Supported)
- 🔄 개발 중 (In Development)
- 📋 계획 (Planned)
- ❌ 폐지 (Deprecated)

### 정보 수집 방법

**기존 문서 분석**:
- product.md에서 제품 기능 추출
- structure.md에서 컴포넌트 목록 추출
- tech.md에서 기술 스택 추출

**소스 코드 분석**:
- 헤더 파일에서 상수/정의 추출
- CMakeLists.txt에서 컴포넌트 목록 추출
- platformio.ini에서 빌드 설정 추출

**웹 UI 확인**:
- 실제 웹 UI에서 설정 옵션 확인
- 가능한 값 범위 확인
- 기본값 확인

**테스트 및 검증**:
- 실제 하드웨어로 기능 검증
- 각 옵션 값 테스트
- 제약 조건 확인

### 버전 관리 전략

**Git 기반 관리**:
- 각 섹션을 별도 파일로 분리
- 변경 시 커밋 메시지에 기능 ID 포함
- 태그로 릴리스 버전 관리

**변경 로그 유지**:
```markdown
## 변경 로그

### v1.2.0 (2024-XX-XX)
- 추가: OBS 프로토콜 지원 (개발 중)
- 개선: LoRa 통신 안정성
- 버그 수정: 웹 UI 연결 문제

### v1.1.0 (2024-XX-XX)
- 추가: 듀얼모드 지원
- 개선: 배터리 효율
```

**소스 코드에서 기능 추출**:
- Python 스크립트로 헤더 파일 파싱
- 상수/정의/열거형 추출

**웹 UI에서 옵션 추출**:

**버전 간 diff 생성**:
- Git diff로 변경된 기능 식별
