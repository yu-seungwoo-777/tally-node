## 개요

이 문서는 기능 카탈로그 편집 시스템의 인수 기준과 테스트 시나리오를 정의합니다. 모든 시나리오는 Given-When-Then 형식을 따릅니다.

---

## 기능 테스트 시나리오

### 시나리오 1: 하드웨어 사양 조회

**설명**: 사용자가 하드웨어 카탈로그에서 특정 사양 조회

**Given** (준비 상태):
- 기능 카탈로그가 게시됨
- 하드웨어 섹션이 존재함
- 사용자가 카탈로그에 접근함

**When** (트리거):
- 사용자가 LoRa 모듈 사양을 검색함

**Then** (예상 결과):
- SX1262 칩셋 정보가 표시됨
- 주파수 대역(923MHz)이 명시됨
- 지원하는 SF/BW/CR 범위가 표시됨
- 최대 출력 전력이 명시됨
- 통신 거리 추정이 포함됨

### 시나리오 2: 소프트웨어 기능 확인

**설명**: 사용자가 특정 소프트웨어 기능의 지원 여부 확인

**Given** (준비 상태):
- 기능 카탈로그가 게시됨
- 소프트웨어 기능 섹션이 존재함

**When** (트리거):
- 사용자가 "OBS 프로토콜" 지원 여부를 확인함

**Then** (예상 결과):
- OBS 프로토콜 항목을 찾을 수 있음
- 상태가 "개발 중"으로 표시됨
- 지원 예정 기능이 나열됨
- 현재 제한 사항이 명시됨
- 관련 프로토콜(ATEM, vMix) 링크 제공

### 시나리오 3: 구성 옵션 범위 확인

**설명**: 사용자가 특정 설정의 가능한 값 확인

**Given** (준비 상태):
- 기능 카탈로그가 게시됨
- 구성 옵션 섹션이 존재함

**When** (트리거):
- 사용자가 LoRa SF(확장 인자) 옵션을 확인함

**Then** (예상 결과):
- SF 범위(7-12)가 명시됨
- 각 값의 특징이 설명됨
- 기본값(SF=9)이 표시됨
- 권장 설정 안내가 포함됨

### 시나리오 4: 버전별 기능 비교

**설명**: 사용자가 펌웨어 버전 간 기능 차이 비교

**Given** (준비 상태):
- 기능 카탈로그가 게시됨
- 버전별 기능 매트릭스가 존재함

**When** (트리거):
- 사용자가 v1.0.x와 v1.1.x의 기능을 비교함

**Then** (예상 결과):
- v1.0.x 기능 목록이 표시됨
- v1.1.x 기능 목록이 표시됨
- 듀얼모드가 v1.1.x에서 추가되었음이 명시됨
- 기능 매트릭스로 시각적 비교 가능
- 마이그레이션 가이드 링크 제공

### 시나리오 5: 프로토콜 지원 기능 상세 확인

**설명**: 사용자가 특정 프로토콜의 상세 지원 기능 확인

**Given** (준비 상태):
- 기능 카탈로그가 게시됨
- 프로토콜 섹션이 존재함

**When** (트리거):
- 사용자가 ATEM 프로토콜의 지원 기능을 확인함

**Then** (예상 결과):
- UDP 포트(9910)가 명시됨
- 최대 채널 수(20)가 표시됨
- 지원 기능 체크리스트가 제공됨
  - [x] Tally 상태 수신
  - [x] 연결 상태 모니터링
  - [x] 자동 재연결
  - [ ] 미디어 클립 정보 (계획)
- 알려진 제한 사항이 명시됨

## 비기능 테스트 시나리오

### 시나리오 6: 카탈로그 완결성 검증

**설명**: 모든 예상 기능이 카탈로그에 포함되어 있는지 검증

**Given** (준비 상태):
- 완성된 기능 카탈로그
- 소스 코드 기능 목록
- 체크리스트

**When** (트리거):
- 검증자가 완결성을 검사함

**Then** (예상 결과):
- 모든 하드웨어 구성 요소가 포함됨
- 모든 소프트웨어 기능이 포함됨
- 모든 지원 프로토콜이 문서화됨
- 모든 구성 옵션이 포함됨
- 누락된 기능이 없음

### 시나리오 7: 일관성 검증

**설명**: 카탈로그 전체의 용어, 형식, 구조 일관성 검증

**Given** (준비 상태):
- 기능 카탈로그
- 일관성 체크리스트

**When** (트리거):
- 리뷰어가 일관성을 검사함

**Then** (예상 결과):
- 기술 용어가 일관되게 사용됨
- 표 형식이 통일됨
- 기능 상태 표시(✅/🔄/📋/❌)가 일관됨
- 섹션 구조가 통일됨
- 링크가 모두 유효함

### 시나리오 8: 정확성 검증

**설명**: 카탈로그 정보가 실제 제품과 일치하는지 검증

**Given** (준비 상태):
- 기능 카탈로그
- 실제 하드웨어
- 최신 펌웨어

**When** (트리거):
- 테스터가 정보를 검증함

**Then** (예상 결과):
- 하드웨어 사양이 실제와 일치함
- 소프트웨어 기능이 실제로 동작함
- 구성 옵션 범위가 정확함
- 프로토콜 지원 현황이 정확함
- 버전 정보가 최신임

## 성공 지표

### 정량 지표

- **커버리지**: 모든 기능 100% 문서화
- **정확도**: 기술적 오류 0개
- **완결도**: 필수 섹션 100% 포함
- **일관도**: 용어/형식 100% 일치

### 정성 지표

- **명확성**: 비전문가도 이해 가능
- **유용성**: 사용자가 필요한 정보를 빠르게 찾음
- **신뢰성**: 실제 제품과 일치
- **유지보수성**: 업데이트가 용이함

### 사용자 피드백

- **찾기 용이성**: 원하는 정보를 찾는 데 걸리는 시간
- **이해도**: 정보 이해 정도
- **만족도**: 카탈로그 만족도 점수
- **지원 요청 감소**: 카탈로그로 해결되는 문의 증가
