## 시작하기

### 시스템 요구사항

- **브라우저**: Chrome 90+, Firefox 88+, Safari 14+
- **네트워크**: WiFi 또는 Ethernet 연결
- **해상도**: 최소 1024x768 (권장 1920x1080)
- **디바이스**: 데스크톱, 태블릿, 모바일 지원

### 접속 방법

1. WiFi 또는 Ethernet으로 Tally Node 네트워크에 연결
2. 브라우저에서 `http://tally-node.local` 또는 `http://192.168.4.1` 접속
3. Web UI가 자동으로 로드됩니다

### 첫 접속 시

초기 설정 마법사가 자동으로 시작됩니다:
1. 네트워크 설정 (AP/STA 모드)
2. 스위처 연결 설정
3. 라이선스 활성화 (선택 사항)

## 대시보드

### 개요

대시보드는 Tally Node 시스템의 전체 상태를 한눈에 확인할 수 있는 메인 페이지입니다. 시스템的健康 상태, 연결 상태, 주요 지표를 실시간으로 모니터링합니다.

### 주요 기능

#### 1. 시스템 상태 카드

**배터리 상태**
- 현재 배터리 잔량을 퍼센트로 표시
- 색상으로 상태 구분:
  - 🟢 50% 이상: 정상 (녹색)
  - 🟡 20-50%: 주의 (노란색)
  - 🔴 20% 미만: 충전 필요 (빨간색)

**온도**
- CPU 온도를 섭씨로 표시
- 정상 범위: 30-60°C
- 주의 범위: 60-70°C
- 경고 범위: 70°C 이상

**업타임**
- 시스템이 실행된 시간을 표시
- 형식: `일:시간:분` 또는 `시간:분`
- 자동 새로고침으로 실시간 업데이트

#### 2. 스위처 상태

**프라이머리 (S1)**
- 연결 상태: 연결됨 / 연결 안 됨
- 스위처 유형: ATEM, vMix
- IP 주소 및 포트
- Tally 상태: PGM/PVW 채널 표시

**세컨더리 (S2)**
- 듀얼 모드 활성화 시 표시
- 프라이머리와 동일한 상태 정보
- 카메라 오프셋 매핑 정보

**듀얼 모드**
- 활성화/비활성화 상태
- 세컨더리 시작 카메라 위치

#### 3. 네트워크 상태

**AP (Access Point)**
- 상태: 활성화/비활성화
- SSID 및 IP 주소
- 연결된 디바이스 수

**WiFi (Station)**
- 연결 상태: 연결됨/연결 안 됨
- SSID 및 IP 주소
- 신호 강도

**Ethernet**
- 하드웨어 감지 상태
- 연결 상태: 연결됨/연결 안 됨
- IP 주소 및 모드 (DHCP/Static)

#### 4. 시스템 정보

**Device ID**
- 고유 디바이스 식별자
- 4자리 16진수 코드

**전압**
- 현재 입력 전압 (V)
- 정상 범위: 11-14V

**버전**
- 펌웨어 버전 정보
- 형식: `major.minor.patch`

### 사용 방법

1. **상태 확인**: 모든 카드의 색상과 텍스트로 전체 상태 파악
2. **실시간 업데이트**: 5초마다 자동으로 데이터 새로고침
3. **빠른 이동**: 각 카드의 "Open →" 버튼으로 해당 설정 페이지로 이동

### 모바일 최적화

모바일 기기에서는 카드가 세로로 배치되어 스크롤하며 확인할 수 있습니다.

## 네트워크 설정

### 개요

네트워크 설정 페이지에서 AP (Access Point), WiFi (Station), Ethernet을 구성합니다. 다양한 네트워크 환경에 맞춰 유연하게 설정할 수 있습니다.

### AP 설정

Access Point 모드는 Tally Node가 WiFi 네트워크를 생성하여 다른 디바이스가 연결할 수 있게 합니다.

#### 설정 항목

**Enable AP**
- 체크박스로 AP 활성화/비활성화
- 비활성화 시 다른 설정은 회색 표시

**SSID**
- 네트워크 이름 (최대 32자)
- 기본값: `Tally-Node-XXXX` (XXXX는 Device ID)
- 영문, 숫자, 특수 문자 가능

**Password**
- WiFi 비밀번호 (최소 8자)
- 눈 아이콘으로 표시/숨김 전환
- 보안: WPA2-Personal

**Channel**
- WiFi 채널 (1-11)
- 권장 채널: 1, 6, 11 (겹침 없는 채널)
- 드롭다운으로 선택

#### 저장 및 적용

1. 설정 입력 후 "Save AP Settings" 버튼 클릭
2. 변경사항이 즉시 적용됩니다
3. AP가 재시작되고 연결된 디바이스가 일시적으로 연결 해제될 수 있습니다

#### 상태 확인

**AP Status** 카드에서 현재 상태 확인:
- State: Enabled/Disabled
- SSID: 현재 네트워크 이름
- Channel: 사용 중인 채널
- IP Address: AP의 IP 주소 (기본: 192.168.4.1)

### WiFi 설정

WiFi Station 모드는 Tally Node가 기존 WiFi 네트워크에 연결합니다.

#### 설정 항목

**Enable WiFi**
- 체크박스로 WiFi 활성화/비활성화

**SSID**
- 연결할 네트워크 이름
- 기존 WiFi 네트워크의 SSID 입력

**Password**
- 네트워크 비밀번호
- 눈 아이콘으로 표시/숨김 전환

#### 저장 및 연결

1. "Save WiFi Settings" 버튼 클릭
2. 자동으로 네트워크 검색 및 연결 시도
3. 연결 성공 시 WiFi Status 카드에 상태 표시
4. 실패 시 비밀번호와 SSID 확인 후 재시도

#### 상태 확인

**WiFi Status** 카드에서 확인:
- Connection: Connected/Disconnected
- SSID: 연결된 네트워크 이름
- IP Address: 할당된 IP 주소

### Ethernet 설정

Ethernet은 가장 안정적이고 빠른 연결 방법입니다.

#### 설정 항목

**Enable Ethernet**
- 체크박스로 Ethernet 활성화/비활성화

**DHCP**
- 토글 스위치로 DHCP/Static 모드 전환
- DHCP: IP 자동 할당 (권장)
- Static: IP 수동 설정

**Static IP 설정** (DHCP 비활성화 시)
- IP Address: 예) 192.168.1.100
- Gateway: 예) 192.168.1.1
- Subnet Mask: 예) 255.255.255.0

#### 저장 및 적용

1. "Save Ethernet Settings" 버튼 클릭
2. 케이블 연결 상태 확인
3. DHCP 모드에서는 IP가 자동으로 할당됩니다
4. Static 모드에서는 입력한 IP로 설정됩니다

#### 상태 확인

**Ethernet Status** 카드에서 확인:
- Hardware: Device Detected/Not Detected
- Connection: Connected/No Link
- Mode: DHCP/Static
- IP Address: 할당된 IP 주소

### 네트워킹 팁

#### 최적 설정 시나리오

**스튜디오 환경**
- AP 모드: 활성화 (디바이스 연결용)
- WiFi: 비활성화
- Ethernet: 활성화 (인터넷 연결용)

**이벤트 현장**
- AP 모드: 활성화
- WiFi: 비활성화
- Ethernet: 비활성화 (케이블 없음)

**테스트/개발**
- AP 모드: 활성화
- WiFi: 활성화 (인터넷 공유)
- Ethernet: 선택사항

#### 문제 해결

**AP에 연결 안 됨**
- SSID와 비밀번호 확인
- 채널을 1, 6, 11 중 하나로 변경
- Tally Node 재시작

**WiFi 연결 실패**
- SSID와 비밀번호 확인
- 네트워크 범위 확인
- 2.4GHz 대역 사용 확인 (5GHz 미지원)

**Ethernet 연결 안 됨**
- 케이블 연결 확인
- 포트 LED 상태 확인
- DHCP 모드로 변경 후 재시도

## 스위처 설정

### 개요

스위처 설정 페이지에서 비디오 스위처 (ATEM, vMix)와 연결하여 Tally 정보를 수신합니다. 듀얼 모드를 사용하여 2개의 스위처를 동시에 연결할 수 있습니다.

### 지원 스위처

- **Blackmagic ATEM**: ATEM Mini, ATEM Mini Pro, ATEM Mini Extreme 등
- **vMix**: vMix 4K, vMix Pro 등

### 프라이머리 스위처 (S1)

#### 설정 항목

**Type**
- 드롭다운에서 스위처 유형 선택
- ATEM 또는 vMix 선택

**Network Interface** (ATEM 전용)
- Auto: 자동으로 네트워크 인터페이스 선택
- Ethernet: Ethernet 포트 사용
- WiFi: WiFi 사용

**IP Address**
- 스위처의 IP 주소
- ATEM 기본값: 192.168.10.240
- vMix: localhost 또는 IP 주소

**Port**
- 연결 포트 번호
- ATEM: 9910 (잠금)
- vMix: 8088 (잠금)
- 자물쇠 아이콘으로 잠금/잠금 해제

**Camera Limit**
- 최대 카메라 수 (0-20)
- 0 = 제한 없음
- 듀얼 모드에서 중요

#### 설정 방법

1. 스위처 전원 켜기
2. Tally Node와 같은 네트워크에 연결
3. 스위처의 IP 주소 확인
4. "Configure" 버튼 클릭
5. 설정 대화상자에 정보 입력
6. "Save" 버튼 클릭

#### 상태 확인

**Connection 상태**
- 🟢 Connected: 연결 성공
- 🔴 Disconnected: 연결 안 됨

**Tally 상태**
- PGM (Program): 현재 방송 중인 카메라
- PVW (Preview): 다음에 방송될 카메라
- 번호로 채널 표시

### 세컨더리 스위처 (S2)

#### 듀얼 모드

듀얼 모드를 사용하면 2개의 스위처를 동시에 연결하여 최대 20개의 카메라를 지원할 수 있습니다.

**활성화 방법**
1. 상단의 "Dual Mode" 토글 스위치 클릭
2. 세컨더리 스위처 카드가 활성화됨
3. "Configure" 버튼 클릭하여 설정

**세컨더리 설정**
- 프라이머리와 동일한 설정 항목
- 독립적인 IP 주소와 연결

### 카메라 매핑

카메라 매핑은 프라이머리와 세컨더리의 카메라 번호를 매핑하여 RX 디바이스에 올바른 Tally를 전송합니다.

#### 매핑 방법

1. **카메라 그리드**: 1-20번 카메라 버튼 표시
2. **색상 구분**:
   - 🔵 파란색: 프라이머리 (S1)
   - 🟣 보라색: 세컨더리 (S2)
   - 🔵🟣 혼합: 두 스위처 모두 사용

3. **세컨더리 시작 위치 설정**:
   - 세컨더리가 시작할 카메라 번호 클릭
   - 예: 5번 클릭 → S2는 카메라 5-20 사용
   - S1은 카메라 1-4 사용

4. **저장**: "Save Mapping" 버튼 클릭

#### 매핑 예시

**표준 구성 (10 카메라)**
- S1: 카메라 1-10
- S2: 사용 안 함
- Offset: 없음

**확장 구성 (20 카메라)**
- S1: 카메라 1-10
- S2: 카메라 11-20
- Offset: 11

**중첩 구성**
- S1: 카메라 1-5
- S2: 카메라 6-15
- Offset: 6

### 연결 테스트

**테스트 방법**
1. 스위처에서 카메라 입력 변경
2. Web UI의 Tally Data 카드 확인
3. PGM/PVW 번호가 올바르게 표시되는지 확인

**연결 문제 해결**
- IP 주소 확인
- 네트워크 연결 확인
- 스위처 전원 확인
- 포트 번호 확인
- 방화벽 설정 확인

### 폴링 속도

**기본 설정**
- 폴링 간격: 5초 (5000ms)
- 변경사항 반영: 최대 5초 지연

**속도 변경**
1. System → Update Speed로 이동
2. 원하는 속도 선택 (1-10초)
3. 변경사항 즉시 적용

**권장 설정**
- 빠른 업데이트 필요: 1-2초
- 표준 사용: 5초 (기본값)
- 네트워크 안정성 우선: 10초

## 방송 채널

### 개요

방송 채널 페이지에서 무선 Tally 신호의 주파수와 Sync Word를 설정합니다. 채널 스캔 기능으로 깨끗한 채널을 찾을 수 있습니다.

### 주파수 설정

#### 주파수 범위

- **범위**: 850-930 MHz
- **간격**: 1 MHz 단위
- **권장**: 868 MHz (한국)

#### 주파수 선택

1. **주파수 슬라이더**: 원하는 주파수로 드래그
2. **빠른 선택**: 하단의 주파수 버튼 클릭
   - 850, 860, 868, 870, 880, 900, 915, 925, 930
3. **현재 주파수**: 중앙에 크게 표시

#### 주파수 변경 주의사항

⚠️ **중요**: 주파수를 변경할 때는 모든 RX 디바이스가 켜져 있어야 합니다

**변경 절차**
1. 모든 RX 디바이스 전원 켜기
2. TX와 RX를 가까운 거리에 두기 (1-2m)
3. 새 주파수 선택
4. "Save Channel Settings" 버튼 클릭
5. 5-10초간 기다리기 (진행률 표시)
6. 모든 디바이스에 새 주파수가 전송됨

**실패 시 재시도**
- 디바이스를 더 가까이 가져오기
- 배터리 충전 확인
- 주파수 재선택 후 재시도

### Sync Word 설정

#### Sync Word란?

Sync Word는 TX와 RX 간의 통신을 식별하는 암호화 키입니다. 같은 Sync Word를 사용하는 디바이스끼리만 통신합니다.

#### 설정 방법

1. **Sync Word 슬라이더**: 1-150 사이 값 선택
2. **16진수 표시**: 0x01-0x96 형식으로 표시
3. **빠른 선택**: 하단의 미리 설정된 값 클릭
   - 0x01, 0x20, 0x40, 0x60, 0x80, 0x96

#### 보안 권장사항

- **기본값**: 0x12 (공통)
- **보안 강화**: 고유한 값 사용 (0x20-0x96)
- **이벤트별**: 이벤트마다 다른 값 사용
- **주의**: TX와 RX가 같은 Sync Word여야 함

### 채널 스캔

#### 스캔 기능

채널 스캔은 850-930 MHz 범위의 모든 채널을 스캔하여 간섭이 없는 깨끗한 채널을 찾습니다.

#### 스캔 방법

1. **시작**: "Start Scan (850-930 MHz)" 버튼 클릭
2. **진행률**: 진행률 표시줄로 스캔 진행 상황 확인
3. **완료**: Scan Results 카드에 결과 표시

#### 스캔 결과

**RSSI Gauge**
- 녹색: 채널이 깨끗함 (Clear)
- 빨간색: 채널이 사용 중 (Busy)
- dBm 값으로 신호 강도 표시

**Recommendation**
- ✅ Good: 현재 채널이 양호함
- ⬆️ Better: 더 좋은 채널 제안
- ⚠️ Change: 채널 변경 권장

**채널 선택**
- 결과 목록에서 채널 클릭
- 자동으로 해당 주파수로 이동
- "Switch to X MHz" 링크로 빠르게 변경

### 현재 채널 상태

**Current Frequency Gauge**
- **Frequency**: 현재 주파수 (MHz)
- **Sync Word**: 현재 Sync Word (0xXX)
- **RSSI Gauge**: 현재 채널의 신호 강도
  - 녹색: 양호
  - 노란색: 보통
  - 빨간색: 나쁨

### 설정 저장

#### 저장 전 체크리스트

- [ ] 주파수 선택 완료
- [ ] Sync Word 설정 완료
- [ ] 모든 RX 디바이스 전원 켜짐
- [ ] TX와 RX 가까운 거리 (1-2m)
- [ ] 배터리 충분 (50% 이상)

#### 저장 방법

1. "Save Channel Settings" 버튼 클릭
2. 진행률 표시줄로 진행 상황 확인
3. 5-10초간 기다리기
4. "Applying settings..." 메시지 표시
5. 완료되면 자동으로 닫힘

#### 저장 확인

- RX 디바이스의 LED가 새 주파수로 변경됨
- Web UI의 Current Frequency Gauge 업데이트됨
- 간섭 없이 Tally 신호 수신됨

### 문제 해결

**RX가 새 주파수를 수신하지 않음**
- 디바이스를 더 가까이 가져오기
- 배터리 충전 확인
- 재시도: 재시작 후 다시 저장

**채널 스캔이 느림**
- 정상 현상: 81개 채널 스캔에 1-2분 소요
- 네트워크 연결 확인

**간섭 계속 발생**
- 다른 주파수로 변경
- Sync Word 변경
- 주변 다른 무선 장치 확인

## 연결된 디바이스

### 개요

연결된 디바이스 페이지에서 모든 RX 디바이스의 상태를 모니터링하고 제어할 수 있습니다. 밝기, LED 색상, 기능 제어가 가능합니다.

### 디바이스 목록

#### 상태 표시줄

**Online/Offline**
- 🟢 Online: 온라인 디바이스 수
- ⚪ Offline: 오프라인 디바이스 수
- **Total**: 전체 디바이스 수 / 라이선스 제한

**Quick Actions**
- **Status 버튼**: 모든 디바이스에 상태 요청 브로드캐스트
- **Brightness 버튼**: 전역 밝기 조절 모달 열기
- **LED Colors 버튼**: LED 색상 프리셋 모달 열기

#### 디바이스 카드

각 디바이스는 개별 카드로 표시됩니다.

**왼쪽 테두리 색상** (신호 강도)
- 🟢 녹색: 양호 (RSSI > -60dBm, SNR > 10dB)
- 🟡 노란색: 보통 (RSSI -60~-70dBm, SNR 5~10dB)
- 🔴 빨간색: 나쁨 (RSSI < -70dBm, SNR < 5dB)
- ⚪ 회색: 오프라인

**디바이스 정보**
- **아이콘**: 온라인 상태에 따른 색상
- **Device ID**: 고유 식별자 (예: A3F2)
- **Camera**: 카메라 번호 배지 (예: CAM 1)
- **Online/Offline**: 연결 상태 배지

**Ping 수치** (온라인만)
- 녹색: < 200ms (양호)
- 노란색: 200-500ms (보통)
- 빨간색: ≥ 500ms (나쁨)
- 🔄 아이콘: Ping 전송 버튼

**배터리 상태**
- **그래프**: 퍼센트로 시각화
- **색상**:
  - 🟢 > 50%: 정상
  - 🟡 20-50%: 주의
  - 🔴 ≤ 20%: 충전 필요
- **텍스트**: 백분율로 표시

**상태 메트릭스**
- **Frequency**: 무선 주파수 (MHz)
- **Sync**: Sync Word (0xXX)
- **Signal**: 신호 강도 (Good/Medium/Weak)

**하단 정보**
- **Uptime**: 디바이스 실행 시간
- **Brightness**: 현재 밝기 (%)

### 디바이스 제어

#### 밝기 제어

**개별 밝기**
1. 카드 하단의 "Brightness" 슬라이더
2. 0-100% 사이 값 조절
3. 변경 사항 즉시 전송

**전역 밝기**
1. 상단의 "Brightness" 버튼 클릭
2. 슬라이더로 밝기 조절 (0-100%)
3. 빠른 설정 버튼: 25%, 50%, 75%, 100%
4. "Close"로 적용 및 닫기

#### 카메라 ID 변경

**변경 방법**
1. 카드 하단의 "Camera ID" 섹션
2. `-` 또는 `+` 버튼으로 조절
3. 또는 숫자 입력으로 직접 입력 (1-20)
4. 변경 사항 즉시 전송

**범위**
- 최소: 1
- 최대: 20
- 0은 허용되지 않음

#### LED 색상 변경

**프리셋 선택**
1. 상단의 "LED Colors" 버튼 클릭
2. 두 가지 프리셋 중 선택:

**Black OFF** (기본)
- PGM (Red): rgb(255, 0, 0)
- PVW (Green): rgb(0, 255, 0)
- OFF: rgb(0, 0, 0)

**Blue OFF**
- PGM (Red): rgb(255, 0, 0)
- PVW (Green): rgb(0, 255, 0)
- OFF: rgb(4, 51, 255) - 파란색

3. 선택된 프리셋에 체크 표시
4. "Apply & Broadcast"로 모든 디바이스에 전송

#### 기능 제어

**Stop/Resume**
- **Stop**: 디바이스 기능 일시 정지
- **Resume**: 정지된 기능 재개
- 버튼 색상과 텍스트로 상태 구분

**Reboot**
- 디바이스 재부팅
- 주의: 재부팅 중에는 연결이 끊김
- 10-20초 후 재연결

**Delete**
- 디바이스 등록 삭제
- 확인 없이 즉시 삭제됨
- 삭제 후 Devices 페이지에서 제거됨

### 디바이스 등록

#### 자동 등록

새로운 RX 디바이스는 자동으로 감지되고 목록에 추가됩니다:

1. RX 디바이스 전원 켜기
2. 같은 주파수와 Sync Word인지 확인
3. 5-10초 내에 Devices 페이지에 나타남
4. 자동으로 Online 상태로 표시됨

#### 수동 등록

자동 등록이 실패한 경우:

1. RX 디바이스 재시작
2. TX와 RX가 같은 주파수와 Sync Word인지 확인
3. "Status" 버튼 클릭으로 브로드캐스트 전송
4. 디바이스가 나타나면 등록 완료

### 디바이스 없음 상태

연결된 디바이스가 없을 때:

- 빈 상태 아이콘 표시
- "No devices connected" 메시지
- "Devices will appear here when they come online" 안내

해결 방법:
1. 주파수와 Sync Word 확인
2. RX 디바이스 전원 확인
3. 배터리 충전 확인
4. "Status" 버튼으로 브로드캐스트

### 문제 해결

**디바이스가 오프라인임**
- 배터리 확인
- 주파수와 Sync Word 확인
- RX 디바이스 재시작
- 거리 확인 (실내 50m, 실외 100m)

**Ping이 높음**
- 거리 단축
- 장애물 제거
- WiFi 간섭 확인

**배터리가 빠르게 소진됨**
- 밝기 낮추기
- 주파수 확인 (높은 주파수는 더 많은 전력 소모)
- 충전 또는 배터리 교체

**밝기/색상 변경 안 됨**
- 디바이스가 온라인인지 확인
- 설정 다시 시도
- 디바이스 재부팅

## 시스템 설정

### 개요

시스템 설정 페이지에서 Tally Node의 전역 설정을 관리합니다. 업데이트 속도, 펌웨어 업데이트, 시스템 재시작 등을 수행할 수 있습니다.

### 업데이트 속도

#### 폴링 속도 설정

**설정 위치**
1. System → Update Speed
2. 드롭다운에서 속도 선택

**속도 옵션**
- **1초**: 가장 빠름 (높은 CPU 사용)
- **2초**: 빠름
- **5초**: 기본값 (권장)
- **10초**: 느림 (낮은 CPU 사용)

#### 권장 설정

**표준 사용**
- 5초 (기본값)
- 대부분의 상황에 적합

**빠른 업데이트 필요**
- 라이브 이벤트
- 1-2초 권장

**네트워크 안정성 우선**
- WiFi 연결
- 10초 권장

**배터리 절약**
- 모바일 사용
- 10초 권장

### 펌웨어 업데이트

#### 업데이트 확인

1. **Check for Updates 버튼**
   - 최신 버전 확인
   - 새 버전이 있으면 다운로드 제안

2. **현재 버전 표시**
   - 현재 펌웨어 버전
   - 형식: major.minor.patch

#### 업데이트 설치

**업데이트 전 준비**
- [ ] 배터리 50% 이상 또는 전원 연결
- [ ] 모든 RX 디바이스가 오프라인
- [ ] 중요한 설정 백업
- [ ] 5-10분 소요 예상

**업데이트 절차**
1. "Download & Install" 버튼 클릭
2. 진행률 표시줄로 진행 상황 확인
3. 자동으로 재시작
4. 업데이트 적용 중이며 3-5분 소요
5. 완료 후 자동으로 재부팅

**업데이트 중 주의사항**
- ⚠️ 전원을 끄지 마세요
- ⚠️ 브라우저를 닫지 마세요
- ⚠️ 네트워크 연결을 유지하세요

### 시스템 재시작

#### 재시작 옵션

**Soft Restart (소프트웨어 재시작)**
- Web UI만 재시작
- 데이터 보존
- 빠름 (5-10초)

**Hard Restart (하드웨어 재시작)**
- 전체 시스템 재시작
- 모든 연결 초기화
- 느림 (30-60초)

#### 재시작 방법

1. "Restart System" 버튼 클릭
2. 재시작 유형 선택 (Soft/Hard)
3. 확인 대화상자에서 "Confirm" 클릭
4. 재시작 진행
5. 완료 후 자동으로 재연결

### 시스템 로그

#### 로그 보기

**로그 레벨**
- ERROR: 오류 메시지만
- WARNING: 경고 및 오류
- INFO: 정보, 경고, 오류 (기본값)
- DEBUG: 모든 메시지

**로그 필터**
- "Network" 네트워크 관련 로그

### 공장 초기화

#### 초기화 옵션

**Network Settings Only**
- 네트워크 설정만 초기화
- AP, WiFi, Ethernet 설정
- 기본값: 192.168.4.1

**All Settings**
- 모든 설정 초기화
- 네트워크, 스위처, 방송 채널
- 라이선스 보존

**Full Factory Reset**
- 모든 설정 및 데이터 초기화
- 라이선스 포함
- 완전히 초기 상태로 복원

#### 초기화 방법

⚠️ **경고**: 초기화는 되돌릴 수 없습니다

1. "Factory Reset" 버튼 클릭
2. 초기화 유형 선택
3. "Confirm Reset" 입력
4. "Reset" 버튼 클릭
5. 30-60초간 초기화 진행
6. 자동으로 재시작

## 부록

### 용어집

- **TX (Transmitter)**: 송신기. Tally 신호를 전송하는 Tally Node
- **RX (Receiver)**: 수신기. Tally 신호를 수신하는 디바이스
- **Tally**: 카메라의 방송 상태 (PGM/PVW)를 표시하는 시스템
- **PGM (Program)**: 현재 방송 중인 카메라
- **PVW (Preview)**: 다음에 방송될 카메라
- **ATEM**: Blackmagic Design의 비디오 스위처 제품군
- **vMix**: PC 기반 비디오 스위처 소프트웨어
- **Sync Word**: TX와 RX 간의 통신 식별 키
- **RSSI (Received Signal Strength Indicator)**: 수신 신호 강도
- **SNR (Signal-to-Noise Ratio)**: 신호 대 잡음비
